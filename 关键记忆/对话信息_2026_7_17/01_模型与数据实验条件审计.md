# 模型与数据实验条件审计

日期：2026-07-17；服务器只读核对，未干扰 v2.2。

## 模型

- 基础 checkpoint 身份匹配官方 `Qwen/Qwen2.5-VL-3B-Instruct`：`model_type=qwen2_5_vl`，架构 `Qwen2_5_VLForConditionalGeneration`，36 层、hidden 2048；视觉塔 32 层、patch 14、spatial merge 2，权重可完整加载。
- 但不是 Qwen 技术报告的复现实验：我们使用 NF4 4-bit QLoRA（r=16、alpha=32、q/v projections）、自定义农业 JSON、200704–401408 pixels、greedy decoding、512 max tokens。
- 当前缺少官方远端 commit/revision 与权重分片对照哈希；正式论文前需冻结模型 revision 和全量 SHA256。

## 数据

- 原始审计：AGE 51,672 图/376 species-stage 类；IP102 classification 75,222 图/102 类；IP102 detection 18,981 图、18,976 标注、22,284 boxes、实际 97 类。
- 当前 Task9/v2.2 实际只使用 IP102 detection 派生数据，没有使用 AGE 的 stage 标签，因此只能支持“97 类害虫候选证据核验/定位”，不能声称完成 102 类、虫态识别或作物病害诊断。
- v2.1 协议有 18,790 families、56,370 rows；正例/语义负例/视觉反事实各 18,790；JSON 四级质量均为 1.0；source hash、near-duplicate component 跨 split 零重叠；query 分布精确匹配；text-only BA/AUROC gate 均 ≤0.55。
- v2.2 微实验：32 类（head/medium/tail=11/11/10），64 train families、32 val、32 dev；split family overlap=0；Control/TaxMask 同 schedule，loss-mass 审计通过。适合验证单一机制，不足以证明全量性能。

## 关键限制

- 当前派生 split 重新划分了 IP102 官方 detection test：3,798 张官方 test 中，2,893 张进入内部 train、369 张进入 val、350 张进入 dev、186 张锁定排除。因此内部结果不能作为官方 IP102 test 成绩；正式基准必须另建严格保留官方 test 的 split。
- AGE 与 IP102 classification 原始审计发现少量跨 split 内容重复；使用前必须按 SHA/near-duplicate component 重新隔离。
- IP102 detection 有 1 个退化框、5 张无标注图、仅 97/102 类有 detection 标注；派生时需保留过滤报告。
- bbox frame 尚未明确贯通原图坐标、Qwen resize 坐标和评测坐标；坐标链审计前不把历史 mIoU 作为可靠结论。
- 当前 real-null 是“真实图像中不存在被查询类”，不等于真实无虫/健康背景；开放世界拒答仍需独立真实负样本和外部数据。

## 结论

- 对当前 v2.2 单变量候选核验实验：**符合条件，但带上述边界条件**。
- 对 Qwen 官方论文/官方 IP102 benchmark 复现：**不符合，不应如此表述**。
- 对完整农业病虫害开放诊断论文：**数据尚不完整**；后续需要独立开放诊断 split、AGE stage 分支、真实无目标样本和跨数据集外部测试。
