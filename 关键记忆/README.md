# 关键记忆使用规范

_固化日期：2026-07-10 | 用途：供后续任务快速恢复项目状态_

---

## 📁 存放规则

- 每天使用一个 `对话信息_YYYY_M_D` 文件夹
- 每个关键事件单独写一个 Markdown 文件，命名为 `NN_简短主题.md`
- 关键事件包括：研究决策、实验结果、数据变化、重要报错与根因、环境配置、论文启发

## 📝 内容规则

- 只记录可复用信息，不复制长篇对话
- 优先写：硬事实、决定、风险、下一步、关联文件
- 实验数据必须同时记录模型版本、数据划分、关键参数和评价指标
- 不确定信息明确标记为“待验证”，不与实验事实混写
- 单份记忆尽量控制在可快速扫描的篇幅内

## 🔎 调用规则

- 新任务开始时，优先读取最近日期文件夹及相关主题记忆
- 新的关键结果产生后，当天立即新增或更新对应记忆文件
- 若新结果推翻旧结论，在新文件中注明被替代的文件与原因
