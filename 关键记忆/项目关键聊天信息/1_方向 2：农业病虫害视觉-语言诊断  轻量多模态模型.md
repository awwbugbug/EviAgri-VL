## 1. 当前领域格局：已经从分类进入“诊断问答 + 解释 + 决策建议”

农业 VLM 方向确实在 2024–2026 明显升温。Agri-LLaVA 构建了约 39.18 万农业病虫害图文/指令数据，覆盖 221 类以上病虫害，并提出农业知识注入训练流程；其 GitHub 也释放了 Agri-400K、Chatbot-Bench 和 VQA-Bench，并支持 LoRA 微调脚本。citeturn988145view0turn988145view1

CDDM/ CDDMBench 则更偏作物病害诊断，包含 13.7 万张作物病害图像、100 万组 QA，覆盖 16 种作物和 60 类病害，并明确把任务从单模态分类扩展到“疾病识别、疾病知识、预防控制策略”的多模态问答。citeturn997317view0turn931122view3

AgroBench 是另一个重要信号：它不是再造训练集，而是做农业 VLM benchmark，覆盖 7 个农业主题、203 个作物类别、682 个病害类别，并由农学专家标注；论文指出现有 VLM 在农业细粒度识别上仍有明显短板，部分开放模型在杂草识别上接近随机水平。citeturn152808search1turn988145view3

更大的 AgriGPT-VL Suite 也已经出现，提出 Agri-3M-VL：100 万图文 caption、200 万图像 grounded VQA、5 万专家级 VQA、1.5 万 GRPO 样本，并建立 AgriBench-VL-4K。这说明“单纯构建一个农业多模态大数据集”已经不是小团队最优路线。citeturn938147search1

**结论：**  
你不能再做“我也微调一个农业 LLaVA，然后比通用 VLM 好一点”。这条路容易被 Agri-LLaVA、CDDM、AgriGPT-VL 压住。更好的切入点是：

> **小模型 + 小训练成本 + 明确图像证据 + 幻觉抑制 + 细粒度虫态/症状知识注入。**

我建议把题目收敛成：

**Evidence-Bound Lightweight Vision-Language Adaptation for Reliable Fine-Grained Crop Pest and Disease Diagnosis**

中文核心：

> 面向作物病虫害细粒度诊断的证据绑定轻量视觉-语言适配方法。

这个题目比原来的 “Lightweight and Explainable Vision-Language Adaptation...” 更强，因为 **Evidence-Bound** 直接把你的核心创新钉死：模型不能只回答，还必须绑定图像局部证据。

# 3. 论文核心问题定义

不要把任务定义成普通 VQA。建议定义成 **Evidence-Bound Agri-Diagnosis**：

输入：

```text
Image I
User question Q
Optional crop/pest prior C
```

输出：

```json
{
  "diagnosis": "rice leaf folder / apple scab / tomato early blight / uncertain",
  "type": "pest | disease | healthy | unknown",
  "fine_grained_attributes": {
    "crop": "...",
    "organ": "leaf / fruit / stem / root",
    "symptom": "spot / chlorosis / necrosis / hole / curling / larva body",
    "color": "brown / yellow / black / white",
    "damage_shape": "round lesion / irregular bite / stripe / mold",
    "pest_stage": "egg / larva / pupa / adult / unknown"
  },
  "visual_evidence": [
    {"region_id": "E1", "bbox_or_mask": "...", "supported_claim": "brown irregular lesion on leaf"}
  ],
  "confidence": 0.78,
  "reliability": "supported | weakly_supported | unsupported"
}
```

这里最关键的是：**回答必须依赖 region evidence**。如果模型说“这是某某病”，但证据区域没有病斑、虫体或损伤形态，就判为 hallucination / unsupported。

# 4. 数据集系统构建

## 4.1 主数据集分层

建议按功能分成四类，而不是简单堆数据。

### A. 害虫细粒度主线：IP102 + 你的多虫态数据

IP102 有 7.5 万张以上图像、102 类害虫，并提供约 1.9 万张检测框标注，天然长尾分布，适合作为“害虫细粒度识别 + 局部证据框”的基础。citeturn997317view1

你的已有优势在这里：

- 你已经做过 IP102；
- 你整理过多虫态监督机制；
- 你可以把“虫态”升级成 VLM 的结构化属性；
- 你可以把分类任务扩展成问答任务。

例如：

```text
Q: What pest stage can be observed in this image?
A: The insect appears to be in the larval stage, supported by the elongated soft body and visible feeding damage on the leaf.
Evidence: E1
```

这比单纯“102 类分类”更有研究价值。

---

### B. 作物病害主线：PlantVillage / PlantVillageVQA / CDDM

PlantVillage 是基础病害分类数据，约 54,303 张健康/病害叶片图像，38 个作物-病害类别，适合做低成本预训练和基础分类 baseline。citeturn997317view2

PlantVillageVQA 是 2025 年基于 PlantVillage 构建的 VQA 数据，包含 193,609 个 QA、55,448 张图像、14 种作物和 38 种病害条件，适合做“病害视觉问答”的标准入门集。citeturn803435academia36

CDDM 更适合做高阶诊断问答，因为它有 13.7 万图像、100 万 QA，问题覆盖疾病识别和防治建议。citeturn997317view0

---

### C. 局部证据主线：PlantSeg + IP102 bbox

PlantSeg 很重要，因为它不是只给类别，而是给病害区域 mask。它包含 11,458 张图像、115 类病害，并带分割标注；论文特别强调其 in-the-wild 场景、病害区域 mask 和复杂背景价值。citeturn997317view3turn997317view4

你的证据绑定模块需要 PlantSeg 这种数据，否则“可解释性”很容易停留在 Grad-CAM 可视化层面，缺少定量证据指标。

---

### D. 泛化测试集：PlantDoc / PlantWild / FieldPlant / AgroBench

PlantDoc 规模较小，2,598 张图像、13 种植物、最多 17 类病害，但它是更接近真实场景的互联网图像，适合作为外部泛化测试集。citeturn803435search2

AgroBench 不一定用于训练，更适合做最终 VLM 泛化能力验证，因为它由专家标注，且覆盖更宽的农业主题。citeturn988145view3

# 5. 建议构建自己的数据格式：EviAgriDiag

不要试图做一个百万级新数据集。你可以做一个中等规模、高质量、证据绑定的数据集，论文价值反而更清晰。

建议命名：

**EviAgriDiag-50K** 或 **AgriEvidenceDiag**

结构如下：

```json
{
  "image": "xxx.jpg",
  "dataset_source": "IP102 / PlantVillage / PlantSeg / CDDM",
  "task_type": "classification / VQA / evidence_qa / hallucination_probe",
  "crop": "rice",
  "target": "rice leaf folder",
  "category_type": "pest",
  "attributes": {
    "organ": "leaf",
    "symptom": "folded leaf and feeding damage",
    "color": "green/brown",
    "shape": "irregular feeding trace",
    "stage": "larva"
  },
  "question": "What pest or disease is shown, and what evidence supports your diagnosis?",

    "bbox": [x1, y1, x2, y2],
```

这个数据集的优势是：**不比规模，比赛题定义和证据质量。**

# 6. 模型系统设计

推荐模型名：

**EviAgri-VL**

全称：

**Evidence-guided Lightweight Agricultural Vision-Language Adapter**

整体结构：

```text
Image
  ├── Global Vision Encoder
  ├── Evidence Region Extractor
  │      ├── Grad-CAM / classifier CAM
  │      ├── SAM / SAM2 mask refinement
  │      └── evidence crop tokens
  ├── Knowledge-Structured Prompt Encoder
  ├── Evidence-Gated Adapter
  └── Frozen / QLoRA-tuned VLM
          ↓
  Evidence-bound diagnosis answer
```

---

## 6.1 Backbone 选择

### 首选 Backbone：Qwen2.5-VL-3B / 7B

Qwen2.5-VL 的优势是本身支持较强图像理解、目标定位、结构化数据抽取，并支持 bounding box / point localization，这和你的“证据绑定”天然契合。citeturn988145view4

建议：

- AutoDL 24GB：优先 Qwen2.5-VL-3B + QLoRA；
- AutoDL 48GB：Qwen2.5-VL-7B + QLoRA；
- 不建议一开始上 72B。

### 第二选择：MiniCPM-V 4.5

MiniCPM-V 4.5 是 8B 级高效 MLLM，论文强调其训练和推理效率，适合作为轻量部署对比 backbone。citeturn590669academia36

### 第三选择：LLaVA-1.5 / LLaVA-OneVision

LLaVA 的好处是生态成熟，visual instruction tuning 体系清楚，复现实验容易；LLaVA 原始工作是把视觉编码器与 LLM 连接起来做通用视觉语言理解。citeturn576877search3turn576877search7

### 强 baseline：Agri-LLaVA

Agri-LLaVA 必须作为农业专用 VLM baseline。它已经有农业病虫害知识注入和 Agri-400K 数据，如果你不和它比，论文说服力会弱。citeturn988145view0turn988145view1

---

## 6.2 轻量微调策略

不要 full fine-tune。主线用 LoRA / QLoRA。

LoRA 的基本思想是冻结预训练权重，在 Transformer 层插入低秩可训练矩阵，从而显著减少可训练参数。citeturn576877search0

QLoRA 则是在 4-bit 量化冻结模型上反传到 LoRA adapter，论文显示可在单张 48GB GPU 上微调 65B 级模型，说明它适合资源受限环境。citeturn576877search1

建议训练参数：

```text
Base VLM: frozen
Vision encoder: frozen or LoRA last blocks only
Projector: trainable
LLM attention q_proj/v_proj: LoRA
Evidence adapter: trainable
Verifier head: trainable
Quantization: 4-bit NF4
```

# 7. 核心创新模块设计

## 模块 1：病虫害结构化知识 Prompt

不是普通 prompt engineering，而是把农业知识变成结构化监督。

字段：

```text
crop        作物
organ       部位：leaf / fruit / stem / root
symptom     症状：spot / necrosis / chlorosis / hole / curling
color       颜色：brown / yellow / black / white / gray
shape       损伤形态：round lesion / irregular lesion / stripe / bite mark
stage       虫态：egg / larva / pupa / adult
risk        危害：feeding / wilting / rot / mildew
control     防治建议
```

形式：

\[
P_k = T(crop, organ, symptom, color, shape, stage, control)
\]

这样你的“多虫态监督机制”可以自然升级为：

> **Pest-stage-aware visual-language diagnosis**

即模型不只识别“是什么虫”，还要回答“处于什么虫态、证据是什么、会造成什么损伤”。

---

## 模块 2：图像局部证据提取

推荐三种路径做消融：

### A. Grad-CAM evidence

Grad-CAM 通过目标类别对最后卷积层的梯度生成类别判别热力图，适合解释 CNN/分类器关注区域。citeturn545288search2

用途：

```text
ConvNeXt/MobileNet classifier → Grad-CAM heatmap → top-k evidence seed
```

### B. SAM / SAM2 mask refinement

SAM 是 promptable segmentation foundation model，可通过点、框、mask 等 prompt 生成分割结果；SAM2 扩展到图像和视频，并在图像分割上比 SAM 更快。citeturn545288search12turn545288search1

用途：

```text
Grad-CAM seed point / IP102 bbox / PlantSeg lesion prior
        ↓
SAM2
        ↓
lesion / pest body mask
```

### C. CAM + SAM2 组合

这是最适合论文的证据提取模块：

\[
H_c = GradCAM(f_{cls}, I, c)
\]

\[
S = Seed(H_c)
\]

\[
M = SAM2(I, S)
\]

其中：

- \(H_c\)：类别 c 的热力图；
- \(S\)：高响应区域种子点或框；
- \(M\)：最终证据 mask。

这个设计比单独 Grad-CAM 更强，因为 CAM 粗，SAM2 可细化边界。

---

## 模块 3：Evidence-Gated Adapter

这是论文方法的核心。不要只是把 mask 图裁剪后喂给模型，而是要做一个 adapter。

设：

- 全局图像特征：\(v_g\)
- 局部证据区域特征：\(r_1, r_2, ..., r_k\)
- 结构化知识 prompt embedding：\(p_k\)

证据融合：

\[
\bar{r} = \frac{1}{K} \sum_{i=1}^{K} r_i
\]

门控权重：

\[
g = \sigma(W_g [v_g; \bar{r}; p_k])
\]

融合视觉 token：

\[
v' = v_g + g \odot W_r \bar{r}
\]

然后把 \(v'\) 输入 VLM projector / adapter。

这个模块的贡献点是：

> 让 VLM 在诊断时显式偏向病斑、虫体、损伤区域，而不是背景、叶片纹理、拍摄环境。

---

## 模块 4：Evidence-Bound Generation

输出时强制模型绑定证据：

```text
Diagnosis: xxx
Evidence: E1 shows brown irregular lesions on the leaf margin.
Reliability: supported
```

训练目标：

\[
L = L_{sft} + \lambda_1 L_{cls} + \lambda_2 L_{attr} + \lambda_3 L_{evid} + \lambda_4 L_{hall}
\]

其中：

- \(L_{sft}\)：普通指令微调 loss；
- \(L_{cls}\)：病虫害类别分类 loss；
- \(L_{attr}\)：症状/虫态/部位属性 loss；
- \(L_{evid}\)：证据区域定位 loss；
- \(L_{hall}\)：幻觉抑制 loss。

如果有 mask：

\[
L_{evid} = 1 - IoU(M, M^*)
\]

如果只有 bbox：

\[
L_{evid} = 1 - \frac{\sum_{(x,y)\in B} H(x,y)}{\sum_{(x,y)} H(x,y)}
\]

也就是热力图能量是否落在真实框内。

---

## 模块 5：幻觉抑制 Verifier

VLM 幻觉在当前多模态模型中是明确问题。2026 年的多模态幻觉综述把 VLM 幻觉定义为生成内容与视觉输入或世界知识不一致，并区分 faithfulness hallucination 与 factuality hallucination。citeturn988145view5

农业诊断里最危险的是三类：

```text
1. 视觉不存在却说存在：没有虫体却说有虫。
2. 属性错配：明明是黄斑却说黑褐色霉层。
3. 防治建议幻觉：诊断不确定却给具体药剂建议。
```

Verifier 做法：

```text
Generated answer
   ↓
claim parser：提取 disease / pest / color / organ / symptom / stage
   ↓
visual evidence classifier：检查图像证据是否支持 claim
   ↓
support score
   ↓
supported / weakly supported / unsupported
```

输出规则：

```text
If support_score < τ:
    answer = "The visual evidence is insufficient for a reliable diagnosis."
```

这会让论文从“准确率提升”转成“可靠农业 AI 诊断”，更容易有研究价值。

```text
Domain keyword coverage
```

但 LLM-as-judge 只能做辅助。已有研究指出农业 VLM 评估方式会显著影响结论，开放式回答的准确率通常比 multiple-choice 低很多，语义 judge 会改变模型排名。citeturn938147academia21

所以主指标要尽量可复现：

```text
label correctness
attribute correctness
evidence correctness
refusal correctness
```

---

## 9.3 证据定位指标

```text
Pointing Game Accuracy
Energy-in-Box
mIoU
Mask Dice
Evidence Recall
Evidence Precision
```

对应数据：

```text
IP102 bbox → Energy-in-Box / Pointing Game
PlantSeg mask → mIoU / Dice
PlantVillage no mask → CAM sanity check only
```

---

## 9.4 幻觉与可靠性指标

建议自定义一个核心指标：

### Evidence-Bound Hallucination Rate, EBHR

\[
EBHR = \frac{N_{unsupported\ claims}}{N_{all\ visual\ claims}}
\]

视觉 claim 包括：

```text
crop
pest/disease
organ
color
symptom
damage morphology
pest stage
```

再做一个：

### Supported Diagnosis Rate, SDR

\[
SDR = \frac{N_{correct\ diagnosis\ with\ correct\ evidence}}{N_{all\ samples}}
\]

这个比 Accuracy 更强，因为它要求：

```text
诊断对 + 证据也对
```

# 11. 可投稿的创新点组合

我建议最后论文贡献写成三点：

## Contribution 1：Evidence-bound agricultural diagnosis task

提出一个农业病虫害视觉-语言诊断任务，要求输出诊断、症状属性和局部视觉证据，而不是只输出类别或自由文本。


比“我提升了 1% accuracy”强很多。

# 15. 主要风险与规避

## 风险 1：数据 QA 是合成的，评审质疑真实性

规避：

```text
只把合成 QA 用作训练；
测试集保留人工规则校验或专家知识模板；
核心指标用 label / attribute / evidence 的客观指标。
```

## 风险 2：SAM2 对小虫体效果不稳定

规避：

```text
害虫用 bbox / detector / CAM；
病斑用 SAM2；
不要强行说一个模块通吃病害和害虫。
```

## 风险 3：防治建议容易幻觉

规避：

```text
防治建议作为 secondary task；
主任务是 diagnosis + evidence；
建议类回答必须在 confidence 足够时输出；
否则输出“需要进一步人工确认”。
```

## 风险 4：Agri-LLaVA / CDDM 已经太强

规避：

```text
不和它们比规模；
比轻量训练成本、证据定位、幻觉抑制、跨数据集可靠性。
```

这个方向可以做，而且比继续卷 IP102 分类更有潜力。最现实、最像论文的路线是：

```text
不要做农业大模型。
不要做普通分类。
不要只做 prompt。
不要只做 Grad-CAM 可视化。

做：

+ 幻觉抑制评估

```

> **Evidence-Bound Lightweight Vision-Language Adaptation for Reliable Fine-Grained Crop Pest and Disease Diagnosis**
