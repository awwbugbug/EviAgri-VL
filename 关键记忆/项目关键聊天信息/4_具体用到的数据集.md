我的提问：ok极限模型和backbone比较清楚了，那具体的数据集我们要用到，或者可能用到哪些？


对，这一步要定清楚。**我们不是把所有农业数据集都塞进去训练**，而是按论文实验功能来选数据集。目标二区的话，数据集要服务于四件事：

1. **诊断准确性**
2. **细粒度虫态/症状属性**
3. **局部证据定位**
4. **跨数据集泛化与幻觉抑制**

我们之前定的主线是“轻量、证据绑定、低幻觉的农业病虫害视觉-语言诊断框架”，数据集也必须围绕这个主线组织，而不是单纯堆规模。

---

# 1. 最推荐的数据集组合

我建议最终论文主数据集体系是：

```text
主训练 / 主评估：
1. 你的 Age / 多虫态害虫数据集
2. IP102
3. PlantVillageVQA 或 CDDM

证据定位评估：
4. PlantSeg
5. IP102 bbox

跨数据集泛化：
6. PlantDoc
7. AgroBench subset

可选增强：
8. Agri-LLaVA / Agri-400K
```

不是所有都必须第一阶段上。最稳的是分阶段。

---

# 2. 第一优先级：你自己的 Age 多虫态数据集

这个是我们最应该抓住的核心资产。

你现在已有的 ages 数据大概是：

```text
约 5 万张图像
约 294 个类别
覆盖 102 种害虫
包含 egg / larva / pupa / adult 等虫态信息
```

这个数据集对我们非常重要，因为它可以支撑别人没有的细粒度任务：

```text
害虫类别识别
虫态识别
species-stage 联合识别
同种不同虫态区分
异种相似虫态区分
虫态证据解释
```

这部分可以直接转成 VLM 数据格式：

```json
{
  "image": "xxx.jpg",
  "task": "pest_stage_diagnosis",
  "question": "What pest and developmental stage are shown in this image?",
  "answer": "The image shows rice leaf folder at larval stage. The diagnosis is supported by the visible soft elongated larval body and feeding damage.",
  "label": "rice_leaf_folder-larva",
  "species": "rice_leaf_folder",
  "stage": "larva",
  "type": "pest"
}
```

这就是我们和普通 PlantVillage 病害分类论文拉开差距的地方。

**它在论文里的作用：**

| 作用            | 说明                              |
| ------------- | ------------------------------- |
| 细粒度害虫诊断主数据    | 支撑 pest diagnosis               |
| 虫态属性监督        | 支撑 stage-aware VLM              |
| 结构化 prompt 来源 | species、stage、morphology        |
| 你已有研究延续       | 从 IP102 / 多虫态监督自然升级到 VLM        |
| 论文差异化         | 别人多做 disease，我们可以做 pest + stage |

这部分建议作为我们的**第一核心数据源**。

---

# 3. 第二优先级：IP102

IP102 必须用。它是害虫方向最标准的公开基准之一。

IP102 包含超过 75,000 张图像、102 类害虫，并且有自然长尾分布；官方还提供约 19,000 张带 bbox 的检测标注。([GitHub][1]) 论文版给出的分类划分是 75,222 张图像、102 类，训练/验证/测试大致为 45,095 / 7,508 / 22,619。([CVF开放获取][2])

**它在我们论文里的作用：**

| 用法             | 具体作用                          |
| -------------- | ----------------------------- |
| 害虫分类 baseline  | 和传统 CNN / ViT / CLIP / VLM 比  |
| 长尾泛化评估         | 用 Macro-F1、Balanced Accuracy  |
| bbox 证据评估      | 用 Energy-in-Box、Pointing Game |
| 和你的 Age 数据衔接   | IP102 原类别 → species-stage 扩展  |
| 支撑 pest branch | 避免论文只做病害，不做害虫                 |

IP102 的 bbox 对我们特别有用，因为我们的论文要讲 **visual evidence**。没有 bbox/mask，证据定位就很难客观评价。

对于 IP102，我们可以做三类任务：

```text
Task 1: Pest Classification
Q: What pest is shown in the image?

Task 2: Pest Evidence Grounding
Q: Which region supports the pest diagnosis?

Task 3: Pest-stage Diagnosis
Q: What pest and developmental stage are shown?
```

第一阶段不用全量 IP102 + 全量 QLoRA。可以先用子集跑通，再全量补实验。

---

# 4. 第三优先级：PlantVillage / PlantVillageVQA

PlantVillage 是病害方向最基础的数据。TensorFlow Datasets 页面显示 PlantVillage 包含 54,303 张健康和病害叶片图像，按物种和疾病分成 38 类。([TensorFlow][3])

但是 PlantVillage 本身是分类数据，不是 VQA 数据。更适合我们的是 **PlantVillageVQA**。

PlantVillageVQA 是 2025 年基于 PlantVillage 构建的 VQA 数据集，包含 193,609 个 QA，配对 55,448 张叶片图像，覆盖 14 种作物和 38 种病害条件，并且问题分成 3 个认知层级和 9 个类别。([arXiv][4])

**它在我们论文里的作用：**

| 数据集                        | 作用                  |
| -------------------------- | ------------------- |
| PlantVillage               | 病害分类 baseline       |
| PlantVillageVQA            | 病害 VQA / 诊断问答训练与测试  |
| PlantVillage → PlantDoc    | 做跨场景泛化              |
| PlantVillageVQA → PlantSeg | 做 evidence transfer |

PlantVillageVQA 很适合做第一版 VLM 微调，因为它已经是 QA 格式，不需要我们从零生成太多问答。

推荐用法：

```text
Train:
PlantVillageVQA train split

Test:
PlantVillageVQA test split

任务：
diagnosis QA
symptom QA
crop/disease verification QA
counterfactual QA
```

它的问题是：PlantVillage 图像比较干净，很多是实验室或较规整背景，**泛化说服力不够**。所以它不能作为唯一数据集。

---

# 5. 第四优先级：CDDM / CDDMBench

CDDM 是更强的病害多模态数据集。它包含 137,000 张作物病害图像和 100 万组 QA，问题覆盖疾病识别、农业知识、防治管理等内容。([arXiv][5])

这个数据集对我们很有吸引力，但要注意两点：

1. **规模大，训练成本高。**
2. **它偏作物病害，不偏害虫虫态。**

所以我建议 CDDM 不作为第一阶段 MVP，而作为第二阶段增强。

**它在论文里的作用：**

| 作用                                | 说明                  |
| --------------------------------- | ------------------- |
| 大规模病害 VQA 训练                      | 增强问答能力              |
| 和 Agri-LLaVA 对齐                   | 说明我们不是小玩具数据         |
| 防治建议任务                            | 可以作为 secondary task |
| CDDM → PlantVillageVQA / PlantDoc | 做泛化                 |
| CDDM subset                       | 控制 AutoDL 成本        |

推荐使用方式：

```text
第一版：
不用全量 CDDM，只抽取 20K–50K QA 做训练/验证。

正式版：
根据资源再扩到 100K–300K QA。

不要一开始上 1M QA。
```

我们论文主线不是“训练大规模农业问答模型”，所以 CDDM 只需要证明我们的框架在大规模病害 VQA 上也成立即可。

---

# 6. 第五优先级：PlantSeg

PlantSeg 对我们的“证据绑定”非常关键。

PlantSeg 是植物病害分割数据集，包含约 11,400 张带病害分割 mask 的图像，覆盖 115 种植物病害，并且主要来自自然环境而不是单纯实验室场景。([arXiv][6]) Kaggle / GitHub 页面也说明其包含超过 11,400 张图像、115 种病害，并提供病害区域 segmentation label。([Kaggle][7])

它在我们论文里的地位很高，因为它能支撑定量证据指标：

```text
mIoU
Dice
Pointing Game
Energy-in-Mask
Evidence Recall
Evidence Precision
```

**它在论文里的作用：**

| 作用            | 说明                           |
| ------------- | ---------------------------- |
| 病斑区域 mask 评估  | 证明 evidence extractor 不是嘴上解释 |
| CAM / SAM2 消融 | Grad-CAM vs SAM2 vs CAM+SAM2 |
| 证据绑定指标        | SDR、EBHR 的视觉依据               |
| 自然场景泛化        | 比 PlantVillage 更真实           |

但是 PlantSeg 不一定适合作为主训练集。它更适合作为：

```text
证据定位评估集
跨数据集测试集
病斑 mask 验证集
```

推荐用法：

```text
Train:
PlantVillageVQA / CDDM

Test:
PlantSeg

评估：
模型是否能把诊断证据落在真实病斑 mask 上。
```

这组实验对论文很重要。

---

# 7. 第六优先级：PlantDoc

PlantDoc 是外部泛化测试集。它规模不大，但场景更真实。

PlantDoc 包含 2,598 张图像，覆盖 13 种植物和最多 17 类病害，图像来自互联网，更接近自然背景和复杂场景。([arXiv][8]) Kaggle 页面也给出相同规模描述。([Kaggle][9])

它不适合做主训练，因为数据太少；但很适合做泛化测试。

**它在论文里的作用：**

| 作用     | 说明                      |
| ------ | ----------------------- |
| 跨数据集泛化 | PlantVillage → PlantDoc |
| 真实背景测试 | 检查模型是否只适应干净叶片图          |
| 幻觉压力测试 | 复杂背景下是否胡说               |
| 外部测试集  | 提高二区论文说服力               |

推荐用法：

```text
Train:
PlantVillage / PlantVillageVQA

Test:
PlantDoc

看：
Accuracy drop
Macro-F1 drop
EBHR increase
Refusal Accuracy
```

如果我们的 EviAgri-VL 在 PlantDoc 上不一定最高，也没关系。关键是看：

```text
unsupported claim 是否更低；
不确定时是否更会拒答；
证据是否更集中在病斑/叶片区域。
```

---

# 8. 第七优先级：Agri-LLaVA / Agri-400K

Agri-LLaVA 是农业 VLM 强相关 baseline。它构建了约 40 万条农业多模态指令数据，覆盖 221 类以上病虫害，并提出农业知识注入训练方法。([arXiv][10]) GitHub 页面说明 Agri-400K 包括农业病虫害特征对齐数据、指令微调数据和评估数据。([GitHub][11])

这个数据集/模型有两个用途：

## 用途 1：作为 baseline

我们需要和 Agri-LLaVA 比：

```text
Agri-LLaVA zero-shot / inference
Agri-LLaVA on its benchmark
Qwen2.5-VL + LoRA
EviAgri-VL
```

## 用途 2：作为可选训练数据

可以抽一部分 Agri-400K 做 SFT，但不建议一开始这么做。

原因：

```text
1. 它数据已经很大，容易把我们变成“复现 Agri-LLaVA”；
2. 它可能和部分公开数据重叠，要注意 train/test leakage；
3. 我们的创新不是数据规模，而是 evidence-bound reliability。
```

所以 Agri-LLaVA 最适合作为：

```text
强 baseline
农业 VLM 领域背景
可选补充训练数据
```

不要让它喧宾夺主。

---

# 9. 第八优先级：AgroBench

AgroBench 是 2025 年的农业 VLM benchmark，覆盖 7 个农业主题，包含 203 个作物类别和 682 个病害类别，并且由农学专家标注。([arXiv][12])

这个数据集更适合做最终外部 benchmark，而不是训练。

**它在论文里的作用：**

| 作用       | 说明                              |
| -------- | ------------------------------- |
| 专家标注外部测试 | 增强论文可信度                         |
| VLM 泛化评估 | 检查通用农业识别能力                      |
| 错误类型分析   | 细粒度 disease / weed / crop error |
| 不做主训练    | 避免任务太散                          |

如果我们后期时间够，可以抽其中 disease identification / pest-related subset 做测试。

推荐写法：

```text
We further evaluate the zero-shot and adapted models on an expert-annotated agricultural VLM benchmark to examine generalization beyond our training datasets.
```

# 11. 最小可行版本：先用哪些？

第一阶段不要贪多。最小可行版本建议：

```text
1. 你的 Age 多虫态数据集
2. IP102
3. PlantVillageVQA
4. PlantSeg
5. PlantDoc
```

这五个就够构成一篇论文原型。

对应关系：

| 实验目标   | 使用数据                                              |
| ------ | ------------------------------------------------- |
| 害虫诊断   | Age + IP102                                       |
| 虫态识别   | Age                                               |
| 病害 VQA | PlantVillageVQA                                   |
| 证据定位   | IP102 bbox + PlantSeg mask                        |
| 跨数据集泛化 | PlantVillageVQA → PlantDoc / PlantSeg             |
| 幻觉测试   | 从 Age / PlantVillageVQA / PlantDoc 构造 conflict QA |

这个版本最现实。

# 12. 标准二区版本：正式论文建议用哪些？

正式论文建议用：

```text
Core:
- Age pest-stage dataset
- IP102
- PlantVillageVQA
- PlantSeg
- PlantDoc

Extended:
- CDDM subset
- Agri-LLaVA inference baseline
```

也就是：

```text
训练：
Age + IP102 + PlantVillageVQA + CDDM subset

证据评估：
IP102 bbox + PlantSeg mask

泛化测试：
PlantDoc + PlantSeg + IP102 held-out / Age held-out

强 baseline：
Agri-LLaVA
Qwen2.5-VL zero-shot
Qwen2.5-VL + vanilla LoRA
```

这已经很够二区了。

# 13. 不建议一开始就用的数据

## 1. 不建议一开始全量 CDDM

100 万 QA 很诱人，但会拖慢实验。

第一阶段只抽：

```text
20K–50K QA
```

足够验证方法。

## 2. 不建议一开始全量 Agri-400K

它会把研究方向带偏，容易变成：

```text
我们用 Agri-LLaVA 数据又训了一个农业 LLaVA
```

这不是我们的主线。

## 3. 不建议找太多零散 Kaggle 病害数据

除非是为了补类别，否则会带来：

```text
类别命名不统一
标签噪声大
train/test 泄漏风险
论文难解释
```

二区论文更看重协议清楚，不是数据越多越好。

# 15. 最终建议

这次数据集不要铺太大。最稳路线是：

```text
第一阶段：
Age + IP102 + PlantVillageVQA + PlantSeg + PlantDoc

第二阶段：
加入 CDDM subset

第三阶段：
加入 Agri-LLaVA baseline / AgroBench subset
```

最终论文可以写成：

> We construct an evidence-bound agricultural diagnosis benchmark from pest, disease, and segmentation datasets, including pest-stage annotations, disease VQA samples, localized evidence masks/boxes, and cross-dataset generalization splits.

中文就是：

> 我们从害虫、病害、分割和 VQA 数据中构建证据绑定农业诊断基准，包含虫态标注、病害问答、局部证据框/掩码和跨数据集泛化划分。

这比简单说“我们用了很多数据集”更有论文价值。
