# 2026 新增顶会顶刊启发与源码审计

日期：2026-07-23  
目的：解释“视觉探针有效、生成式 QLoRA 桥接失败”，为 Task 11B 选择单变量微实验；不改变正在执行的 PlantSeg 审计。

## 核心判断

- 当前病因更像是**视觉信号存在，但语言生成接口没有稳定利用**，不是研究方向或 Qwen 视觉塔整体失效。
- 下一步优先研究冻结 Qwen 内部表示的 evidence/null 可分性，再决定是否训练轻量 evidence head；不再直接扩大 JSON/QLoRA。
- PlantSeg mask 的最大价值是提供病斑局部证据和 disease-without-insect real-null，不应被当作新的害虫分类训练集。

## 冻结后的最小实验候选

仅在 Task11A.3 完成后提出并单独审批：

1. **Task11B-RepProbe（首选）**：同一批 family、同一标签、同一 logistic head，仅比较当前 vision-only pooled feature 与 Qwen 精确 `3L/4` query-token 表示。
2. 图像 token/query token 位置必须由特殊 token 实测定位，禁止照抄 HALP 的比例估计；拆分必须按 family/source，禁止随机图片拆分。
3. synthetic null 只用于 train/val；PlantDoc 与通过人工门控的 PlantSeg 仅作外部 real-null 测试。
4. 门控同时约束：正样本 Macro-F1/coverage、real-null FPR、synthetic-null FPR、source bootstrap CI；不得只按 AUROC 选方案。
5. 只有 query-token 单变量稳定优于 vision-only 且不损害正样本，才授权设计轻量 evidence head。
6. 若不通过，再另立一个单变量 Local-Crop/Mask 实验；不得把 query-token、局部 crop、VEP、VIB 和视觉放大一次叠加。

## 暂不执行

- 不下载新 backbone，不启动 7B，不改动态 LoRA/Gating，不做大规模训练。
- 不直接复制 HALP/ClearSight/VEP/DLCAF 代码；先重写最小、可测试、Qwen2.5-VL-3B 专用实现。
- 不让专家证据包含 GT 类别名、文件路径或 mask 元数据；推理输入仍只能由像素与冻结协议产生。
