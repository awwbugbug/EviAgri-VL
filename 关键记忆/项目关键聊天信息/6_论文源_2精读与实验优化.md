# 论文源_2 精读与实验优化结论

日期：2026-07-16  
范围：Qwen2.5-VL、MMStar、M3ID、AgMMU、HallusionBench、VCD、POPE；结合 Task 9D 法医结果与当前 v2.2 TaxMask 实验。

## 总结判断

- 研究方向没有失败，但任务表述必须收窄：当前 Task 9 实际是“给定候选病虫类别后的证据核验”，不是“从图像独立完成开放诊断”。Task 9D 的高核验 BA 与极低 pest-ID 准确率可以同时成立，不能混成一个结论。
- 更可信的后续框架是：`Diagnose(开放候选生成) → Verify(逐候选视觉核验) → Ground(定位证据) → Abstain(无充分证据则拒答)`。这是待验证路线，不是已证实方法。
- 当前 v2.2 继续按冻结设计完成，严禁中途加入论文模块。论文带来的改进应在其结果之后，以单变量、极小规模实验验证。

## 逐篇结论

| 论文 | 核心发现 | 对本项目的可用价值 | 源码/使用判断 |
|---|---|---|---|
| Qwen2.5-VL | 动态分辨率；图像尺寸按 28 的倍数调整；定位采用输入图像实际尺寸上的绝对坐标，而非默认 0–1000 归一化坐标 | 立刻审计 bbox 坐标链：原图尺寸、Qwen 实际输入尺寸、`image_grid_thw`、GT 坐标系、预测坐标系和逆变换 | 官方 Qwen 仓库 Apache-2.0；工具可用，但当前代码不能仅凭“可运行”判定坐标正确 |
| MMStar | 用无图条件筛掉视觉非必需样本；MG=`S_v-S_wv`，ML=`max(0,S_wv-S_t)` 诊断视觉增益和训练泄漏 | 为农业任务增加 image / no-image 配对分数；项目中的 adapter 文本泄漏指标是“借鉴版”，不能冒充论文原始 ML | 官方仓库有数据和评测代码，但未看到明确 LICENSE；优先重写小型指标实现 |
| M3ID | 用有图/无图下一词分布差异 PDM 衡量视觉条件影响，主要采用 Hellinger 距离；M3ID 在解码时补偿视觉信息随序列增长的衰减 | 先做 teacher-forced PDM-H 法医：按 `evidence_present`、bbox、taxonomy、拒答 token 分组，判断模型究竟在哪些字段使用图像 | 现有 `predictions.jsonl` 只有文本、没有 logits，不能事后直接算 PDM；需另做双前向小样本审计。未找到可信官方源码 |
| AgMMU | 农业多模态基准，强调感知错误与知识错误分离；LoRA 可提升农业问答 | 作为外部迁移评测，而非当前训练数据；必须分别测 image-only、background-only、image+background，并报告视觉增益 | 论文版与当前项目页样本数不同，必须固定版本/revision/hash；论文说明数据 CC-BY-SA 4.0，但仓库代码许可不清晰 |
| HallusionBench | 用原图/编辑图控制对和 figure/pair 一致性定位幻觉；报告准确率、配对正确率、偏置和失败类型 | 将 Task 9 的反事实按 family 成对统计：原图正确且干预图拒答才算因果一致；不能只看总平均 | 官方仓库 BSD-3-Clause；可借鉴 pair/family 评测结构。当前结构化 JSON 不需要 GPT 裁判 |
| VCD | 对原图与扰动图 logits 做对比解码，并用 plausibility constraint 限制候选；可降低物体幻觉 | 仅作为 v2.2 后的推理期基线；需先实现和单测 Qwen2.5-VL 的双视觉流、缓存及 JSON 约束 | 官方仓库 Apache-2.0，但原实现不直接支持 Qwen2.5-VL；约双倍前向开销，且当前 greedy 协议不能偷换为 sampling |
| POPE | 将幻觉转为平衡的物体存在性问答；负类分 random/popular/adversarial；报告 Accuracy/Precision/Recall/F1/Yes ratio | 可构造 Agri-POPE：精确类别边界、family split、随机/高频/共现困难负类；定位为“候选存在性核验”，不替代开放诊断 | 官方仓库 MIT，可复用采样和指标思想；原版名称匹配解析器不宜原样用于本项目 |

## 对现有实验最重要的修正

1. **bbox 坐标协议存在未排除风险。** 当前协议将源标注 bbox 直接写入答案；推理经过动态 resize；提示没有声明原图绝对坐标、模型输入坐标或归一化坐标；评估又直接做 IoU。现有 mIoU 不能在坐标链审计前被当作可靠定位能力。
2. **PDM 不是简单 log-prob 比值。** 应按论文用有图/无图条件分布距离（优先 Hellinger）衡量视觉依赖。现有输出无 logits，必须另做只读双前向审计。
3. **VCD 不是可直接插入的开关。** 官方实现面向其他模型/旧 QwenVL；移植到 Qwen2.5-VL 要保持同一解码协议并验证 structured JSON，不能以改变 sampling 的方式获得不公平结果。
4. **AgMMU 有版本漂移。** 论文、GitHub README 与当前项目页规模不一致；使用时必须记录下载日期、commit/revision、文件哈希和实际样本数。
5. **验证任务与开放诊断必须分开。** POPE/MMStar/HallusionBench能强化“是否有证据”的科学性，却不能证明模型能自主识别病虫类别。

## 冻结后的最小实验路线

### P0：v2.2 完成后立即做，不训练新模型

1. **BBox coordinate audit（优先级最高）**：抽取 32 个含框样本，记录原图 W/H、实际模型输入 W/H、`image_grid_thw`、GT frame；做已知人工框 round-trip 与三种坐标解释的 IoU 对照。若坐标不一致，先修评测协议，历史定位分数标记为不可解释。
2. **Family/pair 因果指标**：为现有 dev 输出补算“原图正确且 blank/blur/shuffle 拒答”的 joint success、pair consistency、干预前后具体诊断下降率，继续禁止读取 Task 8 locked set。
3. **PDM-H 小审计**：同一批 32 families，原图/无图双前向，teacher-forced 计算各字段 token 的 Hellinger 距离；不改权重、不训练、不做 M3ID 解码。

### P1：依据 v2.2 结果分流

- 若 TaxMask H1 通过：再做一个诊断头与核验头解耦的微型单变量实验，先证明候选来自像素而非 prompt。
- 若 H1 不通过：停止扩大复合 JSON 训练；先建立独立的开放诊断小任务（图像→Top-k pest candidates），核验任务保持为第二阶段。
- VCD 仅在 Qwen2.5-VL 移植通过单测后，以 Base/最佳 adapter 的同协议配对推理作为基线；不立即做 DPO 或训练期融合。

### P2：协议稳定后

- 构建 Agri-POPE-Standard（随机/高频/困难共现负类）和 Agri-POPE-Causal（blank/blur/shuffle/crop/换图），按 real-null 与 synthetic-null 分开报告。
- AgMMU 只做固定版本外部评测：image-only、background-only、image+background；用视觉增益区分农业知识与图像感知。
- 只有 v2 方案、超参数和决策规则全部冻结，才一次性运行 Task 8 的 186-family confirmatory audit。

## 预注册的新假设

- H-tax：TaxMask 在不改变正负 loss mass 的条件下，提升独立 taxonomy 预测而不损害核验可靠性。
- H-coord：统一 bbox 坐标链后，定位指标可复现且显著高于坐标错配基线。
- H-visual：正确原图在关键诊断/证据 token 上的 PDM-H 高于 blank/shuffle/no-image。
- H-decode：同协议 VCD 降低 null 具体诊断率，同时不使正样本 Accuracy 下降超过 3pp。
- H-external：在 AgMMU 上 image+background 明显优于 background-only，证明收益不只是文本农业知识。

## 当前明确不做

- 不干扰或修改运行中的 v2.2；不读取 Task 8 locked 186 families。
- 不进入全量 9E、动态 LoRA/Gating、SAM2、7B、新 backbone、DPO。
- 不因论文指标好看就一次叠加 PDM、VCD、POPE 数据和新结构；所有新增因素保持单变量、小样本、可回滚。
